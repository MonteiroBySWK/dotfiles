**Objetivo:**
Você é um assistente especialista em engenharia de software. Sua tarefa é analisar o documento de requisitos em anexo e converter as funcionalidades, tarefas, histórias de usuário e bugs descritos em um arquivo de formato CSV. Este CSV será usado por um script para importar e criar páginas automaticamente em um banco de dados do Notion.

**Formato e Regras do CSV:**

Você deve gerar o conteúdo do CSV seguindo estritamente as especificações abaixo.

**1. Cabeçalho do CSV:**
A primeira linha do arquivo deve ser o cabeçalho, exatamente como a seguir:
`title,status,tipo,prioridade,especialidade,epicoId,responsavelId,who,what,for,acceptanceCriteria,technicalTasks`

**2. Definição das Colunas e Regras de Preenchimento:**

* `title`: (Texto) O título principal do item, que **deve seguir o formato** `PREFIXO-XX [Especialidade] - Descrição`.
    * **PREFIXO**: Use `US` para `User Story`, `TSK` para `Task`, `BUG` para `Bug`, e `SPK` para `Spike`.
    * **XX**: Um número sequencial para identificar o item (ex: 01, 02, 03...).
    * **[Especialidade]**: O valor exato da coluna `especialidade` entre colchetes.
    * **Descrição**: O nome ou resumo descritivo do item.
    * **Exemplos de Títulos Finais:**
        * `US-01 [Frontend] - Cadastro de Candidatos`
        * `TSK-02 [Backend] - Criar endpoint de validação de CPF`
        * `BUG-03 [Frontend] - Botão de salvar não responde no formulário`

* `status`: (Seleção) O status inicial do item. Deve ser **exatamente um** dos seguintes valores:
    * `Impedimento`
    * `Backlog`
    * `A fazer`
    * `Em Progresso`
    * `Concluído`
    * `Em andamento`
    *(Geralmente, para novos itens, use "Backlog" ou "A fazer", a menos que o documento especifique o contrário).*

* `tipo`: (Seleção) O tipo do item. Deve ser **exatamente um** dos seguintes valores:
    * `Spike`
    * `Task`
    * `Bug`
    * `User Story`

* `prioridade`: (Seleção) O nível de prioridade. Deve ser **exatamente um** dos seguintes valores:
    * `Muito Alta`
    * `Alta`
    * `Média`
    * `Baixa`

* `especialidade`: (Seleção) A área ou time responsável. Deve ser **exatamente um** dos seguintes valores:
    * `Backend`
    * `Frontend`
    * `DevOps`

* `epicoId`: (ID de Relação) O ID da página do "Épico" pai. Se esta informação não estiver no documento, deixe a célula **vazia**.

* `responsavelId`: (ID de Relação) O ID do "Responsável" pela tarefa. Se não houver um responsável definido, deixe a célula **vazia**.

* `who`, `what`, `for`: (Texto da User Story)
    * Estas colunas correspondem ao formato da história de usuário: "**Como** `who`, **Quero** `what`, **Para** `for`".
    * Preencha estas colunas **apenas se** o `tipo` for `User Story`. Para os outros tipos (`Task`, `Bug`, `Spike`), deixe estas três colunas **vazias**.
    * **Importante:** Se qualquer um desses campos contiver vírgulas, ponto e vírgula, ou outros caracteres especiais, o campo inteiro deve ser envolvido em aspas duplas para evitar erros de parsing no CSV. Por exemplo: `"equipe técnica"`, `"criar, editar e gerenciar PIAs"`.

* `acceptanceCriteria`: (Lista de Texto)
    * Liste todos os critérios de aceitação para a tarefa ou história.
    * **Regra Crítica:** Múltiplos critérios devem ser separados pelo caractere pipe (`|`).
    * **Importante:** O campo completo deve ser envolvido em aspas duplas se contiver vírgulas, pipes, ou outros caracteres especiais. Isso garante que a lista seja tratada como um único campo. Exemplo: `"Usuário consegue fazer login com credenciais válidas|Sistema exibe mensagem de erro para login inválido|Campo de senha é mascarado"`.

* `technicalTasks`: (Lista de Texto)
    * Liste todas as tarefas técnicas necessárias para a implementação.
    * **Regra Crítica:** Múltiplas tarefas devem ser separadas pelo caractere pipe (`|`).
    * **Importante:** Sempre envolva este campo em aspas duplas, pois ele contém pipes e pode conter vírgulas. Exemplo: `"Criar endpoint de autenticação|Desenvolver UI da tela de login|Escrever testes de integração para o fluxo de login"`.

**Instruções Finais:**

* **Aderência Estrita:** Siga rigorosamente os valores permitidos para as colunas de seleção (`status`, `tipo`, `prioridade`, `especialidade`) e o formato da coluna `title`.
* **Dados Ausentes:** Se uma informação solicitada por uma coluna não estiver presente no documento de requisitos, deixe a célula correspondente **vazia**. Não presuma ou invente dados.
* **Manuseio de Campos com Caracteres Especiais:** Para evitar erros de parsing no CSV, todos os campos de texto que contenham vírgulas, ponto e vírgula, pipes, ou quaisquer caracteres que possam interferir com os delimitadores CSV devem ser envolvidos em aspas duplas. Isso se aplica principalmente às colunas `who`, `what`, `for`, `acceptanceCriteria`, e `technicalTasks`.
* **Saída:** Sua resposta deve conter **apenas o conteúdo do CSV**, começando pela linha de cabeçalho. Não inclua explicações ou formatação adicional. Certifique-se de que cada linha tenha exatamente 12 colunas, com campos vazios representados por nada (ou seja,,,) e campos preenchidos devidamente escapados com aspas se necessário.

Agora, por favor, processe o documento de requisitos em anexo e gere o conteúdo CSV.
